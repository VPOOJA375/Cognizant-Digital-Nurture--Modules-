# Cognizant Digital Nurture Program - Weeks 1 & 2

## Overview

This repository contains the solutions and implementations completed during Weeks 1 and 2 of the Cognizant Digital Nurture Program. The exercises focus on Design Patterns, Data Structures & Algorithms, PL/SQL Programming, JUnit 5, Mockito, Spring Core, Maven, and Spring Data JPA concepts.

## Week 1 - Exercises Completed

### 1. Singleton Pattern

* Implemented the Singleton Design Pattern.
* Ensured that only one instance of the Logger class is created.
* Demonstrated controlled access to a shared object.

### 2. Factory Method Pattern

* Implemented the Factory Method Design Pattern.
* Created different document types using factory classes.
* Demonstrated loose coupling and object creation abstraction.

### 3. E-Commerce Platform Search Function

* Implemented product search functionality.
* Used Linear Search to locate products by ID.
* Analyzed search efficiency and time complexity.

### 4. Financial Forecasting

* Implemented a recursive solution for financial forecasting.
* Calculated future values based on growth rates.
* Demonstrated the use of recursion in problem-solving.

### 5. PL/SQL Control Structures

* Used IF, ELSIF, and ELSE statements.
* Implemented decision-making logic based on conditions.
* Displayed results using DBMS_OUTPUT.

### 6. PL/SQL Stored Procedures

* Created and executed stored procedures.
* Updated employee salary records using procedure parameters.
* Demonstrated code reusability in database programming.

## Week 2 - Exercises Completed

### 1. JUnit 5 - Setting Up JUnit

* Configured a JUnit 5 testing environment.
* Created unit tests using the `@Test` annotation.
* Verified application functionality using assertions.

### 2. JUnit Assertions

* Used `assertEquals()`, `assertTrue()`, `assertFalse()`, `assertNull()`, and `assertNotNull()`.
* Validated expected and actual outputs during testing.

### 3. Arrange-Act-Assert (AAA) Pattern

* Implemented the Arrange-Act-Assert testing pattern.
* Used `@BeforeEach` and `@AfterEach` for test setup and cleanup.
* Improved test organization and maintainability.

### 4. Mockito - Mocking and Stubbing

* Created mock objects using Mockito.
* Used `when()` and `thenReturn()` to simulate dependency behavior.
* Tested service classes independently.

### 5. Mockito - Verifying Interactions

* Verified method invocations using `verify()`.
* Ensured expected interactions between objects during testing.

### 6. Spring Core - Basic Application Configuration

* Configured a basic Spring application using XML configuration.
* Managed beans using the Spring IoC Container.

### 7. Spring Core - Dependency Injection

* Implemented Constructor-based Dependency Injection.
* Demonstrated loose coupling between application components.

### 8. Maven Project Configuration

* Created and configured a Maven project.
* Managed project dependencies using `pom.xml`.

### 9. Spring Data JPA - Quick Example

* Developed a simple Spring Data JPA application.
* Performed basic CRUD operations using JPA repositories.

### 10. JPA, Hibernate & Spring Data JPA

* Understood the relationship between JPA, Hibernate, and Spring Data JPA.
* Learned their roles in enterprise Java application development.
